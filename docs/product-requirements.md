# プロダクト要求定義書

## プロダクトビジョンと目的

Pinboard のパブリックブックマークから、注目記事を毎日自動でピックアップし、要約付きメールで届けるキュレーションサービス。

**目的：** Webフロントエンド分野の最新動向を優先しつつ、手間なく・毎朝・効率よく話題の記事をキャッチアップする。

## 主要な機能一覧

1. Pinboard パブリックフィードからの記事収集
2. OpenAI による記事の興味スコアリングと日本語要約生成
3. 重複排除（既送信記事を除外）
4. 毎朝9時（JST）のメール自動配信

## 機能要件

### 記事収集

- Pinboard のパブリック JSON popularフィード（`https://feeds.pinboard.in/json/popular/`）を使用する
- 1回の収集でフィードからデフォルト件数（50件）の記事を取得する
- 実装前にフィードの稼働状況と返却件数を実機確認すること

### フィルタリング・ランキング

2段階フィルタで Cloudflare Workers Free プランのサブリクエスト上限（50件/呼び出し）内に収める。

#### 第1段階：事前スコアリング（サブリクエスト不要）

- Pinboardフィードの title / description を使い、OpenAI GPT-4o-mini でWebフロントエンド関連度をスコアリングする
- 上位20〜30件を候補として選出する

#### 第2段階：記事フェッチ＋最終スコアリング

- 候補記事のURLにアクセスし、`<title>` / `<meta name="description">` / 記事本文テキスト（先頭3,000文字）を取得する
- 取得した情報をもとに OpenAI GPT-4o-mini で再スコアリングし、フロントエンド関連の上位記事を選出する
- 英語記事も対象とする（要約は日本語で生成）
- 取得した本文テキストは要約生成にも使用する

#### フォールバック：件数補充

- 第2段階でフロントエンド関連記事が5件未満の場合、不足分をフロントエンド以外の人気記事（スコア上位）で補充し、合計5件にする
- 補充記事はメール内で区別せず同列に扱う

#### スコアリング基準（システムプロンプトに記述）

高評価の基準：

- JavaScript / TypeScript の新機能・ベストプラクティス
- React / Next.js / Vue などのフレームワーク
- CSS・ブラウザAPI・パフォーマンス最適化
- ツールチェーン（Vite, esbuild 等）

低評価・除外の基準：

- マーケティング・採用・SEO目的の記事
- バックエンド・インフラ中心の内容

### 重複排除

- 送信済み記事のURLをデータベースに記録する
- 過去に送信した記事は選出対象から除外する
- 保持期間：90日間

### メール配信

- 配信先：環境変数 `TO_EMAIL` で設定する（Cloudflare Workers シークレット）
- 配信時刻：毎朝9時（JST / UTC+9）
- 送信元：Resend を使用（`resend.dev` デフォルトドメイン、`onboarding@resend.dev`）
- 形式：プレーンテキスト
- 件名：`[Daily Pinboard] YYYY-MM-DD のフロントエンド記事`

### メール本文の構成

```text
件名: [Daily Pinboard] 2026-05-07 のフロントエンド記事

本日のWebフロントエンド注目記事 5選です。

---

1. 記事タイトル
   URL: https://...
   要約: （2〜3文の日本語要約。記事の要点と読む価値を伝える）

2. 記事タイトル
   ...

---
```

> 配信停止・変更は `TO_EMAIL` 環境変数を更新することで対応する（返信受信の仕組みは持たない）。

## 非機能要件

### コスト

- 月額 $1 以下を目標とする
- 有料サービスは OpenAI API（GPT-4o-mini）のみ
- Cloudflare Workers / D1 / Cron Triggers：無料枠内で運用
- Resend：無料プラン（3,000通/月）を使用

### 可用性・信頼性

- 記事取得失敗時はリトライを1回行う
- フォールバック補充後も5件に満たない場合は取得できた件数で送信する（0件の場合はスキップ）
- エラー時のサイレント失敗を避け、ログに記録する
- エラーログは Cloudflare Workers のログ（`wrangler tail`）で確認する。永続的なエラー検知が必要になった場合は D1 への記録を追加で検討する

### 要約品質

- 要約は日本語で2〜3文とする
- 記事の要点と読む価値が伝わる内容にする
- 事実と異なる情報を含まないようシステムプロンプトで明示的に指示する

### テスト品質

- **テスト駆動開発（TDD）を採用する。** 実装前にテストを書く
- すべてのビジネスロジック関数に単体テスト（Vitest）を書く
- CI でテストが失敗した場合はデプロイをブロックする

### セキュリティ

- APIキー類はすべて Cloudflare Workers の環境変数（シークレット）で管理する
- ソースコードにキーをハードコードしない

### 技術スタック

- **実行環境：** Cloudflare Workers
- **スケジューラ：** Cloudflare Cron Triggers（`0 0 * * *` UTC = 毎朝9時 JST）
- **データベース：** Cloudflare D1
- **AI：** OpenAI API（GPT-4o-mini）
- **メール送信：** Resend
- **デプロイ：** Wrangler CLI / GitHub Actions
