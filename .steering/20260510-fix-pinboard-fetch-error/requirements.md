# 要求内容：Pinboard JSON API フェッチエラーの修正

## 概要

Pinboard の JSON フィード（`https://feeds.pinboard.in/json/popular/`）からデータを取得する際に断続的なエラーが発生している。現在の実装には複数の脆弱な点があり、本作業でこれらを修正する。

## 発生しているエラー

Cloudflare Workers のログに以下のエラーが記録されている（cron: `0 0 * * *`）。

```text
Error: HTTP error: 503
  at retryFetch (index.js:99:21)
  at async fetchFeed (index.js:111:15)
  at async Object.scheduled (index.js:287:17)
```

Pinboard が 503（Service Unavailable）を返した際に、リトライが尽きてクラッシュしている。

## 現状の問題

`src/fetcher.ts` の `retryFetch` 関数に以下の問題がある。

なお、HTTP エラーやネットワークエラーに対するリトライ処理（最大 2 回）は既に実装されている。

### 問題 1：リトライ間に待機がない（主因）

503 は一時的なサーバー不調を示すステータスコードであり、少し待てば回復することが多い。しかし現在の実装はリトライ間に一切の待機がなく、即座に再リクエストするため Pinboard が回復する前に同じ 503 を受け取り続けてリトライを使い果たす。

### 問題 2：`res.json()` のパースエラーがリトライ対象外

`retryFetch` は HTTP 層のエラーのみをリトライする。`res.json()` の呼び出しはリトライロジックの外にある。Pinboard が一時的に不正な JSON や空レスポンスを返した場合、リトライなしに即座にクラッシュする（今回のエラーとは別の潜在的障害点）。

### 問題 3：レスポンス構造の検証なし

パース後のデータが期待する配列形式（`PinboardItem[]`）であるかを検証していない。Pinboard がエラーオブジェクト（例：`{"error": "..."}` ）を返した場合、`.map()` で実行時エラーになる（今回のエラーとは別の潜在的障害点）。

## ユーザーストーリー

- Daily Pinboard の定期実行が、Pinboard API の一時的な不調によって不必要に失敗しない
- フェッチエラーが発生した場合、適切な待機を挟んでリトライし、回復可能なエラーは自動復旧する
- リトライ後も失敗する場合は、明確なエラーメッセージをスローする

## 受け入れ条件

- [ ] リトライ間にエクスポネンシャルバックオフ（または固定の待機）を実装する
- [ ] `res.json()` のパースエラーもリトライ対象に含める
- [ ] パース後のデータが配列であることを検証し、不正な場合はエラーをスローする
- [ ] 既存のテストがすべて通過する
- [ ] 新しい挙動に対応したテストを追加する

## 制約事項

- Cloudflare Workers 環境（`setTimeout` は使用不可。`await new Promise(resolve => setTimeout(resolve, ms))` は Workers でも動作するが、確認が必要）
- 外部ライブラリの追加は最小限に抑える
- Pinboard API の URL やレスポンス形式は変更しない
