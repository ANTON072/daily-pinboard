# 新機能の追加

実装する機能: $ARGUMENTS

以下の手順で新機能を実装してください。

## 1. プロジェクト理解

- CLAUDE.mdを読んで、プロジェクトの全体像を把握する。
- `docs/architecture.md` を確認して、レイヤー構造を理解する。
- `docs/repository-structure.md` を確認して、ファイル配置ルールを把握する。

## 2. ステアリングファイルの作成

CLAUDE.mdの「機能追加・修正時の手順」に従い、以下を順に作成する。
各ファイルを作成するたびに内容を提示し、承認を得てから次に進む。

```bash
mkdir -p .steering/[YYYYMMDD]-[開発タイトル]
```

1. `requirements.md` - 要求内容（機能の説明・受け入れ条件・制約事項）
2. `design.md` - 設計（実装アプローチ・変更コンポーネント・影響範囲）
3. `tasklist.md` - タスクリスト（具体的な実装ステップと完了条件）

## 3. 既存パターンの調査

- Grepを使って類似する既存機能を検索する。
- 見つかった実装ファイルを読み、以下を確認する:
  - コンポーネントの構造・命名規則
  - ファイル配置ルール
  - 状態管理・データフェッチのパターン
  - テストの書き方

## 4. 実装

- `tasklist.md` のタスクに従って実装を進める。
- `architecture.md` のレイヤー構造を遵守する。
- 既存パターンに合わせた命名・ファイル配置を守る。

## 5. テストと品質チェック

- テストを実行する。
- `pnpm biome check` でリント・フォーマットを確認する。
- エラーが出た場合:
  - `architecture.md` を再確認
  - 既存パターンとの差異を確認
  - 修正して再テスト

## 6. ドキュメント更新

- 変更が基本設計に影響する場合は `docs/` 内の該当ドキュメントを更新する。
- `tasklist.md` のタスクを完了済みに更新する。
